# Casos de uso do Terraform

O [HashiCorp Terraform](/terraform/intro) é uma ferramenta de infraestrutura
como código que permite definir recursos de infraestrutura em arquivos de
configuração legíveis por pessoas, que podem ser versionados, reutilizados e
compartilhados.
Você pode então usar um fluxo de trabalho consistente para provisionar e
gerenciar sua infraestrutura com segurança e eficiência ao longo de todo o seu
ciclo de vida.

Esta página descreve casos de uso populares do Terraform e fornece recursos
relacionados que você pode usar para criar configurações e fluxos de trabalho do
Terraform.

## Implantação em multi-nuvem

O provisionamento de infraestrutura em várias nuvens aumenta a tolerância a
falhas, permitindo uma recuperação mais eficiente em caso de interrupções do
provedor de nuvem.
No entanto, as implantações em multi-nuvem adicionam complexidade, pois cada
provedor tem suas próprias interfaces, ferramentas e fluxos de trabalho.
O Terraform permite que você use o mesmo fluxo de trabalho para gerenciar vários
provedores e lidar com dependências entre nuvens.
Isso simplifica o gerenciamento e a orquestração de infraestruturas de grande
escala em multi-nuvem.

### Recursos

- Faça nosso tutorial
  [Implante clusters Kubernetes federados em multi-nuvem](/terraform/tutorials/networking/multicloud-kubernetes)
  para provisionar clusters Kubernetes em ambientes Azure e AWS, configurar a
  federação do Consul com gateways de malha entre os dois clusters e implantar
  microsserviços entre os dois clusters para verificar a federação.
- Navegue pelo
  [Registro do Terraform](https://registry.terraform.io/browse/providers) para

## Ferramentas para implantação, escalabilidade e monitoramento de infraestrutura de aplicações

Você pode usar o Terraform para implantar, lançar, escalar e monitorar a
infraestrutura de aplicações de várias camadas com eficiência.
A arquitetura de aplicações em N camadas permite escalar os componentes da
aplicação de forma independente e proporciona uma separação de
responsabilidades.
Uma aplicação pode consistir em um conjunto de servidores web que usam uma
camada de banco de dados, com camadas adicionais para servidores de API,
servidores de cache e malhas de roteamento.
O Terraform permite gerenciar os recursos em cada camada de forma conjunta e
lida automaticamente com as dependências entre elas.
Por exemplo, o Terraform implantará uma camada de banco de dados antes de
provisionar os servidores web que dependem dela.

### Recursos

- Faça nosso tutorial
  [Automatize o monitoramento com o provedor Datadog do Terraform](/terraform/tutorials/applications/datadog-provider)
  para implantar uma aplicação Nginx de demonstração em um cluster Kubernetes
  com o Helm e instalar o agente Datadog em todo o cluster.
  O agente Datadog reporta a integridade do cluster para o seu dashboard do
  Datadog.
- Faça nosso tutorial
  [Use balanceadores de carga de aplicações para implantações azul-verde e canário](/terraform/tutorials/aws/blue-green-canary-tests-deployments).
  Você irá provisionar os ambientes azul e verde, adicionar opções de
  ativação/desativação de recursos à sua configuração do Terraform para definir
  uma lista de possíveis estratégias de implantação, realizar um teste canário e
  promover incrementalmente seu ambiente verde.

## Clusters de autoatendimento

Em uma grande organização, seu time de operações centralizado pode receber
muitas solicitações de infraestrutura repetitivas.
Você pode usar o Terraform para criar um modelo de infraestrutura de
"autoatendimento" que permite que os times de produto gerenciem sua própria
infraestrutura de forma independente.
Você pode criar e usar módulos do Terraform que codificam os padrões para
implantação e gerenciamento de serviços em sua organização, permitindo que os
times implantem serviços de forma eficiente, conforme as práticas da sua
organização.
O HCP Terraform também pode ser integrado a sistemas de tickets como o
ServiceNow para gerar automaticamente novas solicitações de infraestrutura.

### Recursos

- Faça o tutorial
  [Use módulos do registro](/terraform/tutorials/modules/module-use) para
  começar a usar módulos públicos em sua configuração do Terraform.
- Faça o tutorial
  [Crie e use um módulo local](/terraform/tutorials/modules/module-create) para
  criar um módulo para gerenciar buckets do AWS S3.
- Siga estas [Instruções de configuração da integração do Service Catalog do ServiceNow](/terraform/cloud-docs/integrations/service-now)
  para conectar o ServiceNow ao HCP Terraform.

## Conformidade e gerenciamento de políticas

O Terraform pode ajudar você a aplicar políticas sobre os tipos de recursos que
os times podem provisionar e usar.
Os processos de revisão baseados em tickets são um gargalo que pode atrasar o
desenvolvimento.
Em vez disso, você pode usar o Sentinel, um framework de política como código,
para aplicar automaticamente políticas de conformidade e governança antes que o
Terraform faça alterações na infraestrutura.
As políticas do Sentinel estão disponíveis no Terraform Enterprise e no
[HCP Terraform](https://www.hashicorp.com/products/terraform/pricing).

### Recursos

- A
  [documentação de estimativa de custos](/terraform/cloud-docs/cost-estimation)
  descreve como usar o recurso de estimativa de custos e como definir políticas
  que limitem os custos associados a alterações na infraestrutura.
- A [documentação do Sentinel](/terraform/cloud-docs/policy-enforcement) fornece
  informações mais detalhadas e uma lista de políticas de exemplo que você pode
  adaptar aos seus casos de uso.

## Configuração de aplicações PaaS

Plataformas como Serviço (PaaS) como o Heroku permitem que você crie aplicaçÕes
web e adicione complementos, como bancos de dados ou provedores de e-mail.
O Heroku pode escalar elasticamente o número de dynos ou workers, mas a maioria
das aplicações mais complexas exige muitos complementos e serviços externos.
Você pode usar o Terraform para codificar a configuração necessária para uma
aplicação Heroku, configurar um DNSimple para definir um CNAME e configurar o
Cloudflare como uma Rede de Distribuição de Conteúdo (CDN) para a aplicação.
O Terraform pode fazer tudo isso de forma rápida e consistente, sem a
necessidade de uma interface web.

### Recursos

Faça o tutorial
[Implante, gerencie e escale uma aplicação no Heroku](/terraform/tutorials/applications/heroku-provider)
para gerenciar o ciclo de vida de uma aplicação com o Terraform.

## Redes definidas por software

O Terraform pode interagir com Redes Definidas por Software (SDNs) para
configurar automaticamente a rede conforme as necessidades das aplicações em
execução.
Isso permite migrar de um fluxo de trabalho baseado em tickets para um
automatizado, reduzindo os tempos de implantação.

Por exemplo, quando um serviço se registra no
[HashiCorp Consul](https://www.consul.io/), o
[Consul-Terraform-Sync](/consul/docs/nia) pode gerar automaticamente a
configuração do Terraform para expor as portas apropriadas e ajustar as
configurações de rede para qualquer SDN que tenha um provedor Terraform
associado.
A Automação de Infraestrutura de Rede (NIA) permite aprovar com segurança as
alterações necessárias para suas aplicações, sem precisar traduzir manualmente
os tickets das pessoas desenvolvedoras nas alterações que você considera
necessárias para as aplicações.

### Recursos

- Faça o tutorial
  [Introdução à automação de infraestrutura de rede com Consul-Terraform-Sync](/consul/tutorials/network-infrastructure-automation/consul-terraform-sync-intro)
  para instalar o Consul-Terraform-Sync em um nó.
  Em seguida, você o configurará para se comunicar com um datacenter do Consul,
  reagir a alterações de serviço e executar uma tarefa de exemplo.
- Faça o tutorial
  [Consul-Terraform-Sync e integração Terraform Enterprise/Cloud](/consul/tutorials/network-infrastructure-automation/consul-terraform-sync-terraform-enterprise)
  para configurar o Consul-Terraform-Sync para interagir com o Terraform
  Enterprise e o HCP Terraform.

## Kubernetes

O Kubernetes é um agendador de cargas de trabalho de código aberto para
aplicações conteinerizadas.
O Terraform permite implantar um cluster Kubernetes e gerenciar seus recursos
(por exemplo, pods, deployments, serviços, etc.).
Você também pode usar o
[Operador HCP Terraform](https://github.com/hashicorp/hcp-terraform-operator)
para gerenciar infraestrutura em nuvem e local por meio de uma Definição de
Recurso Personalizado (CRD) do Kubernetes e do HCP Terraform.

### Recursos

- Faça o tutorial
  [Gerencie recursos do Kubernetes via Terraform](/terraform/tutorials/kubernetes/kubernetes-provider).
  Você usará o Terraform para agendar e expor uma implantação do NGINX em um
  cluster Kubernetes.
- Faça o tutorial
  [Implante infraestrutura com o Operador HCP Terraform para Kubernetes](/terraform/tutorials/kubernetes/kubernetes-operator).
  Você configurará e implantará o Operador em um cluster Kubernetes e o usará
  para criar um espaço de trabalho do HCP Terraform e provisionar uma fila de
  mensagens para uma aplicação de exemplo.

## Ambientes paralelos

Você pode ter ambientes de staging ou de controle de qualidade que usa para
testar novas aplicações antes de lançá-las em produção.
À medida que o ambiente de produção cresce e se torna mais complexo, pode ser
cada vez mais difícil manter um ambiente atualizado para cada etapa do processo
de desenvolvimento.
O Terraform permite que você crie e desative rapidamente infraestrutura para
desenvolvimento, teste, QA e produção.
Usar o Terraform para criar ambientes descartáveis conforme a necessidade é mais
econômico do que manter cada um indefinidamente.

## Demonstrações de software

Você pode usar o Terraform para criar, provisionar e inicializar uma
demonstração em vários provedores de nuvem.
Isso permite que as pessoas usuárias finais experimentem facilmente o software
em sua própria infraestrutura e até mesmo ajustem parâmetros como o tamanho do
cluster para testar as ferramentas de forma mais rigorosa em qualquer escala.
